# Руководство продукт-менеджера
## Система управления дебиторской задолженностью

---

## Что это за система

Веб-приложение для коллекторских агентов и их руководителей. Позволяет вести базу должников, управлять делами о задолженности, рассылать уведомления через несколько каналов (SMS, WhatsApp, Telegram, Email) и отслеживать эффективность работы агентов.

---

## Навигация (боковое меню)

```
Основное
├── Дашборд          — ключевые метрики и графики
├── Дела             — список дел о задолженности
├── Должники         — база должников
└── Обещания (PTP)   — обещания должников об оплате

Операции
├── Уведомления      — шаблоны и рассылки
├── Расписание       — запланированные рассылки
└── Отчёты           — аналитика кампаний и агентов

Администрирование
├── Пользователи     — управление агентами и доступами
└── Интеграции       — настройка каналов связи
```

## Страницы и флоу

### 1. Дашборд `/dashboard`

**Что показывает:**
- **Открытые дела** — сколько дел сейчас в работе, из них сколько просрочено
- **Общий долг** — суммарная задолженность по всем открытым делам (в сомах) и процент сбора
- **Обещания (PTP)** — сколько обещаний ожидают исполнения, сколько выполнено в этом месяце
- **Уведомлений сегодня** — сколько сообщений отправлено за сегодня по всем каналам
- **Столбчатый график** — динамика кампаний за последний период
- **Круговая диаграмма** — распределение дел по статусам

**Флоу:** зайти → посмотреть общую картину → при необходимости перейти в нужный раздел по ссылкам в карточках.

---

### 2. Должники `/debtors`

Центральный справочник физических лиц, которым предъявлены требования.

**Что можно делать:**

| Действие | Как |
|---|---|
| Найти должника | Поиск по ФИО или номеру телефона (в реальном времени, с задержкой 400 мс) |
| Посмотреть детали | Кликнуть на строку → открывается боковая панель |
| Создать должника | Кнопка «Добавить» → форма |
| Импортировать список | Кнопка «Импорт» → загрузить файл |
| Редактировать | В боковой панели → иконка карандаша |
| Удалить | В боковой панели → иконка корзины → подтверждение |

**Боковая панель должника содержит:**
- ФИО, телефон
- Контакты: email, WhatsApp, Telegram
- Список дел о задолженности этого должника (с суммой, статусом, DPD) → каждое дело — кликабельная ссылка

**Каналы связи** отображаются цветными бейджами в строке таблицы (WhatsApp, Telegram, Email). Если бейдж есть — контакт заполнен и рассылка по этому каналу возможна.

---

### 3. Дела о задолженности `/debt-cases`

Каждое «дело» — это конкретный долг конкретного должника, назначенный на агента.

**Ключевые понятия:**
- **Сумма** — сумма долга в сомах
- **DPD** (Days Past Due) — количество дней просрочки. Подсвечивается красным при DPD > 60, оранжевым при > 30
- **Дата погашения** — срок, к которому должник обязан был погасить долг. Показывает «просрочено N дн.» или «сегодня»
- **Статус** — текущее состояние дела (новое / в работе / обещание / закрыто / просрочено)
- **Агент** — кто ведёт дело

**Что можно делать:**

| Действие | Как |
|---|---|
| Поиск | По имени должника, с задержкой |
| Фильтр по статусу | Выпадающий список статусов |
| Сортировка | Клик на заголовки «Сумма», «DPD», «Дата погашения» |
| Создать дело | Кнопка «Создать дело» |
| Изменить дело | В строке → меню «⋯» → «Изменить» |
| Удалить | В строке → меню «⋯» → «Удалить» → подтверждение |
| Открыть детали | Клик на имя должника в строке |
| Сменить статус | Прямо в таблице — выпадающий список в колонке «Статус» |

#### Страница детального дела `/debt-cases/{id}`

Открывается при клике на имя должника в таблице.

**Содержит:**
- Карточки: сумма долга, DPD, дата погашения, назначенный агент
- Контакты должника
- Вкладка **Уведомления** — история отправок по этому делу + кнопка «Отправить уведомление»
- Вкладка **PTP** — список обещаний об оплате по этому делу + кнопка «Добавить PTP»
- График истории DPD

**Отправка уведомления из дела:**
1. Открыть дело → вкладка «Уведомления» → «Отправить уведомление»
2. Выбрать канал (SMS, WhatsApp, Telegram, Email) и шаблон
3. Нажать «Отправить» → уведомление уходит немедленно

**Добавление PTP из дела:**
1. Открыть дело → вкладка «PTP» → «Добавить PTP»
2. Указать дату обещания и обещанную сумму
3. Сохранить

### 4. Обещания об оплате `/ptp`

PTP (Promise to Pay) — фиксация обещания должника заплатить к определённой дате.

**Что можно делать:**

| Действие | Как |
|---|---|
| Фильтр по статусу | Таб-переключатель: Все / Ожидает / Выполнено / Нарушено |
| Добавить PTP | Кнопка «Добавить PTP» → выбрать дело из списка, указать дату и сумму |
| Отметить выполненным | В строке → кнопка «Выполнено» (зелёная, только для «Ожидает») |
| Отметить нарушенным | В строке → кнопка «Нарушено» (красная, только для «Ожидает») |

**Индикаторы сроков** в колонке «Дата обещания»:
- «просрочено N дн.» — красный, дата уже прошла
- «сегодня» — жёлтый
- «через N дн.» — жёлтый, если ≤ 7 дней

> PTP можно создать как с этой страницы (выбрав дело), так и прямо из страницы конкретного дела.

### 5. Уведомления `/notifications`

Управление шаблонами сообщений и ручная рассылка.

Страница разбита на **4 вкладки:**

#### Вкладка «Шаблоны»
Список шаблонов для рассылок. Шаблон — это текст с переменными (например, `{{full_name}}`, `{{amount}}`).

**Что можно:**
- Создать шаблон (кнопка «Новый шаблон»): название, канал (SMS/WhatsApp/Telegram/Email), текст с переменными
- Предпросмотр шаблона с подстановкой реальных значений
- Просмотреть список существующих шаблонов

#### Вкладка «Рассылка»
Массовая рассылка по шаблону.

**Флоу:**
1. Выбрать шаблон
2. Выбрать канал
3. Опционально — выбрать конкретные дела (иначе рассылка по всем)
4. Нажать «Разослать» → система отправляет всем должникам

#### Вкладка «SMS (Nikita)»
Прямая отправка SMS через сервис Nikita SMSPro (nikita.kg) без шаблона.

**Флоу:** указать номер телефона → написать текст → отправить.

#### Вкладка «Лог отправок»
История всех отправленных уведомлений.

**Фильтры:** по каналу (SMS/WhatsApp/Telegram/Email), по статусу (В очереди / Отправлено / Доставлено / Ошибка).

**Статусы доставки:**
- **В очереди** — сообщение ещё не отправлено
- **Отправлено** — ушло из системы
- **Доставлено** — подтверждено провайдером
- **Ошибка** — не удалось отправить

### 6. Расписание `/scheduler`

Запланированные рассылки — отправятся автоматически в указанное время.

**Что можно делать:**

| Действие | Как |
|---|---|
| Создать задачу | «Запланировать» → выбрать шаблон, канал, дату и время |
| Фильтр по статусу | Таб-переключатель: Все / Ожидает / Отправлено / Отменено / Ошибка |
| Отменить задачу | В строке → «Отменить» → **диалог подтверждения** → «Да, отменить» |

**Индикаторы времени** в колонке «Время отправки»:
- «через N мин.» — если до отправки меньше часа (жёлтый)
- «через N ч.» — если от 1 до 24 часов

> Отменить можно только задачи в статусе «Ожидает». Уже отправленные и отменённые — нельзя.

---

### 7. Отчёты `/reports`

Аналитика в двух вкладках.

#### Вкладка «Кампании»
Эффективность рассылок за период.

**Фильтры:** диапазон дат (с/по), канал. Быстрые пресеты: «Сегодня», «7 дней», «30 дней», «Месяц».

**Таблица показывает по каждому дню и каналу:**
- Отправлено / Доставлено / % доставляемости (зелёный ≥ 80%, жёлтый ≥ 50%, красный < 50%)
- Количество PTP, полученных после рассылки
- Собрано сомов

В нижней строке таблицы — **итоговая сумма** по всем строкам.

**Экспорт:** кнопка «Экспорт CSV» — скачивает данные таблицы в файл.

#### Вкладка «Активность агентов»
Лог действий каждого агента: что делал, когда, с каким делом.

**Экспорт:** кнопка «Экспорт CSV».

### 8. Пользователи `/users`

Управление учётными записями агентов и администраторов.

**Что можно делать:**

| Действие | Как |
|---|---|
| Найти пользователя | Поиск по логину или email (ищет по всем пользователям) |
| Посмотреть детали | Кликнуть на строку → боковая панель |
| Создать пользователя | «Добавить» → логин, email, пароль, роли (опционально) |
| Редактировать | Открыть боковую панель → иконка карандаша → изменить email / пароль / роли |
| Удалить | В боковой панели → иконка корзины → подтверждение |

**Боковая панель пользователя** показывает: логин, email, дату создания, назначенные роли.

**Роли** определяют права доступа. Назначаются при создании или редактировании через чекбоксы.

> Пустое поле «Новый пароль» при редактировании = пароль не меняется.

### 9. Интеграции `/integrations`

Настройка внешних сервисов для отправки уведомлений.

**Поддерживаемые каналы:**

| Канал | Провайдер | Что нужно |
|---|---|---|
| SMS | Nikita SMSPro (nikita.kg) | Логин, пароль, имя отправителя |
| WhatsApp | Chat2Desk (chat2desk.com) | API-токен |
| Telegram | Telegram Bot API | Токен бота |
| Email | SMTP | Хост, логин, пароль |

**Что можно делать:**

| Действие | Как |
|---|---|
| Добавить интеграцию | «Добавить» → выбрать канал, заполнить параметры |
| Проверить соединение | На карточке интеграции → «Тест» → результат появится под кнопкой |
| Отправить напрямую (Chat2Desk) | На карточке → «Отправить» → указать номер и текст |
| Удалить | На карточке → иконка корзины → подтверждение |

Статус интеграции: **Активна** (зелёный бейдж) или **Неактивна** (серый).

## Ключевые термины

| Термин | Расшифровка |
|---|---|
| **DPD** | Days Past Due — количество дней просрочки платежа |
| **PTP** | Promise to Pay — зафиксированное обещание должника заплатить к конкретной дате |
| **Кампания** | Массовая рассылка уведомлений по группе дел |
| **Рассылка** | Одна волна уведомлений по шаблону через выбранный канал |
| **Доставляемость** | % сообщений, которые были подтверждены провайдером как доставленные |
| **Собрано** | Сумма долгов, погашенных после рассылки за период |
| **Шаблон** | Текст уведомления с переменными `{{full_name}}`, `{{amount}}` и т.д. |

## Типичные сценарии работы

### Сценарий 1: Новый должник и дело
1. **Должники** → «Добавить» → заполнить ФИО, телефон, email/WhatsApp/Telegram
2. **Дела** → «Создать дело» → выбрать должника, указать сумму и срок погашения, назначить агента
3. Готово — дело появилось в списке со статусом «Новое»

### Сценарий 2: Массовая рассылка должникам
1. **Уведомления** → вкладка «Шаблоны» → убедиться, что нужный шаблон есть (или создать новый)
2. Вкладка «Рассылка» → выбрать шаблон, канал → нажать «Разослать»
3. Через несколько секунд сообщения уйдут должникам по выбранному каналу
4. Прогресс доставки — в **Лог отправок**

### Сценарий 3: Запланировать рассылку на утро
1. **Расписание** → «Запланировать»
2. Выбрать шаблон, канал, указать дату и время (например, завтра 09:00)
3. Сохранить — задача появится со статусом «Ожидает»
4. В указанное время рассылка уйдёт автоматически

### Сценарий 4: Должник позвонил и обещал заплатить
1. **Дела** → найти нужное дело → кликнуть на должника
2. На странице дела → вкладка «PTP» → «Добавить PTP»
3. Указать дату (когда обещает заплатить) и сумму
4. В день дедлайна агент заходит в **Обещания** → отмечает «Выполнено» или «Нарушено»

### Сценарий 5: Посмотреть эффективность за месяц
1. **Отчёты** → вкладка «Кампании»
2. Нажать «Месяц» (быстрый пресет) → «Применить»
3. Таблица покажет: сколько сообщений отправлено, % доставки, сколько PTP, сколько собрано
4. При необходимости — «Экспорт CSV» → открыть в Excel

### Сценарий 6: Добавить нового агента
1. **Пользователи** → «Добавить»
2. Указать логин, email, пароль
3. Отметить нужные роли
4. Нажать «Создать» — агент может войти в систему с этими данными

## Статусы дел

| Статус | Когда используется |
|---|---|
| **Новое** | Дело только создано, работа не начата |
| **В работе** | Агент активно работает с должником |
| **Обещание** | Должник дал PTP |
| **Просрочено** | Срок погашения прошёл, долг не погашен |
| **Закрыто** | Долг погашен или дело закрыто по другой причине |

Статус можно сменить прямо в таблице дел через выпадающий список в колонке «Статус».
